Internet控制报文协议

ICMP简介

  • ICMP（Internet Control Messages Protocol）

    • IP提供不可靠、无链接、尽最大努力交付分组的服务，若出现异常，则要通知源站。如：路由器找不到目的站或下一跳路由器；分组TTL为0；网络出现拥塞。

  • IP自身没有差错报告和纠正机制，使用了网络层的ICMP协议（RFC792）

    ICMP的封装

  • ICMP数据封装在IP数据报里传输，IP首部协议字段值为1。

  • 注：-ICMP报文可以被IP协议、用户进程使用。

    -ICMP是IP协议的补充，不能看成高层协议。

    ICMP的报文格式

    首部和数据部分

  • ICMP数据报由8字节的首部和可变长度的数据部分组成。

  • 前4字节对所有的报文类型都是相同的，有三个字段：类型，代码和校验和。

  • 后4个字节的内容与ICMP的类型有关。

    各字段的含义

  • 类型（type）：8位，描述ICMP报文的类型。

  • 代码（code）：8位，进一步区分类型中的几种不同情况。

  • 校验和：16位，进行差错校验，覆盖整个ICMP报文。

    ICMP的报文类型

    差错报告报文

  • 报告路由器或主机正在处理IP数据报遇到的问题，其数据部分携带引起差错的原始数据报的一部分。

    查询报文

  • 一般成对出现，帮助网络管理员或者主机从一个路由器或者另外一个主机得到特定的消息，其数据部分懈怠了基于查询类型的额外信息。

    ICMP查询报文

    作用

    可以获得特定主机或者路由的信息，对某些网络问题进行诊断。

    回显请求和应答报文

  • 用来确定两个节点之间在IP这一级能否正常通信。

  • 类型为8表示回显请求报文，类型为0表示回显应答报文。

  • 标识符字段
    用于标识ICMP报文，成对的请求和应答报文使用相同的表示字段。

  • 序列号字段

    保证了请求与应答报文的一一对应，即一个request包与其reply包的序列号是一样的。

ping命令

  • 产生多个回显请求或回显应答报文，用于检查另一个主机是否可达，测试两个站点在IP层的连通性。
  • 网络不通时，常先ping本机，再ping网关，然后ping DNS服务器。

时间戳请求和应答报文

  • 某些应用（或者分布式系统）需要各设备的时钟大致同步。

  • 某些应用则希望估算两台主机之间传输数据的往返时间（比如往返时间设置为等待回应的超时时间间隔）。

  • 为了满足上述要求，ICMP定义了时间戳请求报文（timestamp request）和时间戳应答报文（timestamp reply）。

  • 类型13：时间戳请求报文；类型14：时间戳应答报文

  • 每个时间戳字段占32位，存放一个整数，表示从世界时(格林尼治时间)的午夜到当前时刻的毫秒数。

  • 处理过程

    1. 源点在时间戳请求报文的原始时间戳字段填入它的时钟所显示的通用时间。其他两个字段都是零。
    2. 终点把请求报文中的原始时间戳字段值复制到应答报文的同一个字段中去。
    3. 然后在接收时间戳字段中填入收到的这个请求报文时其时钟所显示的通用时间。
    4. 终点在应答报文将要发送时在发送时间戳字段中填入其时钟所显示的通用时间

  • 计算时延

    设原始时间戳为T1，接收时间戳为T2，发送时间戳为T3，原发送方收到回应时间为T4，则：

    往返时延：T4-T1；

    单程时延：[T4-T1-(T3-T2)]/2，其中(T3-T2)为接收方的处理时延。

ICMP差错报告报文

作用

  • 如果在投递IP数据报的过程中发生了错误，则需要使用ICMP差错报文向源结点报告差错。
  • ICMP不能纠正差错，它只是报告差错，差错纠正留给高层协议去做。
  • ICMP差错报告报文是封装在IP数据报中传输的，同样会出现差错，会被路由器丢弃。

发送规则

  • 差错报文的数据字段包括出错分组的首部和前8个字节的数据。
    • 首部包含了IP控制信息，数据的前8字节提供了端口号（tcp或udp）和序号（tcp)的信息，可以根据这些信息通知上层协议。
  • 仅向数据包的源端报告差错（不包括中间的路由），由源端采取相应措施。
  • 携带ICMP差错报告报文的IP数据不再产生ICMP差错报告报文。
  • 分片的IP数据报，如果不是第一个分片，这部产生ICMP差错报文。
  • 具有多播地址的分组不产生ICMP差错报告报文。
  • 具有特殊地址的分组（如127.0.0.0或0.0.0.0）不产生差错报告报文。

五种差错报告报文

ICMP校验和

  • ICMP校验和的计算覆盖了整个ICMP报文。
  • ICMP计算校验和的方法与IP首部校验和的计算方法相同。
