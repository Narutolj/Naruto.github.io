概述

概念

Extensible Markup Language 可扩展标记语言

  • 可扩展：标签都自定义的。<user>、<student>

功能

存储数据

  • 配置文件
  • 在网络中传输

XML与HTML的区别

  1. xml标签是自定义的，html标签是预定义
  2. xml的语法严格，html语法松散
  3. xml是存储数据的，html是展示数据的

语法

基本语法

  1. xml文档的后缀名为 .xml
  2. xml第一行必须定义为文档声明
  3. xml文档中有且仅有一个跟标签
  4. 属性值必须使用引号引起来
  5. 标签必须正确关闭
  6. xml的标签必须区分大小写

快速入门

组成部分

  1. 文档声明

    • 格式：<?xml 属性列表 ?>
    • 属性列表：
      • version：版本号，必须的属性
      • encoding：编码方式。告知解析引擎当前文档使用的字符集，默认值：ISO-8859-1
      • standalone：是否独立
        • 取值：yes（不依赖其他的文件）、no（依赖其他的文件）

  2. 指令

    <?xml-stylesheet type=”text/css” href=”a.css” ?>

  3. 标签

    XML 元素必须遵循以下命名规则：

    • 名称可以包含字母、数字以及其他的字符
    • 名称不能以数字或者标点符号开始
    • 名称不能以字母 xml（或者 XML、Xml 等等）开始
    • 名称不能包含空格

    可使用任何名称，没有保留的字词。

  4. 属性

    id值唯一

  5. 文本

    CDATA区：在该区的数据会被原样展示

    格式：<![CDATA[数据]]>

约束

规定xml文档的书写规则

分类

  1. DTD：一种简单的约束技术
  2. Schema：一种复杂的约束技术

DTD

  1. 引入DTD文档到xml文档中

    • 内部的DTD文档：将约束规则定义在xml文档中

    • 外部的DTD文档：将约束规则定义在外部的DTD的文件中

      本地:<!DOCTYPE 根标签名 SYSTEM “dtd文件的位置”>

      文件:<!DOCTYPE 根标签名 PUBLIC “dtd文件名字” “DTD文件的位置URL”>
